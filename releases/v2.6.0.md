# v2.6.0 - ESLint 10, AI Search API Migration, CI/CD Hardening

## Highlights

- **ESLint 10 Migration** — Upgraded tooling from ESLint 9 to ESLint 10 with full codebase remediation
- **AI Search API Migration** — Transitioned from deprecated `autorag()` to new `aiSearch()` Cloudflare API
- **Docker CI/CD Hardening** — Tag-only triggers, jq-based version detection, removed VERSION file
- **P111 Exact Pinning** — Override dependencies use exact pins to prevent lockfile drift
- **6 Security Fixes** — minimatch ReDoS, tar path traversal vulnerabilities patched

## Changed

- **ESLint 10 Migration**: Upgraded eslint 9.39.2 → 10.0.3, fixed 14 violations across codebase
- **AI Search API Migration**: `autorag()` → `aiSearch()` with backwards-compatible response adapter
- **Default Branch Migration**: Renamed `master` → `main` across all workflow files

## Security

- **GHSA-3ppc-4f35-3m26** (minimatch ReDoS): Resolved all 11 npm audit vulnerabilities via ESLint 10 upgrade
- **GHSA-7r86-cg39-jmmj / CVE-2026-27903 / CVE-2026-27904** (minimatch ReDoS): Updated to 10.2.4
- **CVE-2026-26960** (tar path traversal): Updated tar override 7.5.7 → 7.5.8
- **P111 Exact Pinning**: glob, tar, minimatch overrides now use exact version pins

## CI/CD

- Docker publish triggers only on tag pushes (prevents duplicate builds)
- VERSION file removed; version read from package.json via jq with fail-on-error
- Removed Dependabot auto-merge workflow
